BMS Digital Card parametrization

Description of the set of parameters for the Digital Card in the HIL Connect Interface component.

Real-time only: This document is valid only for real-time simulation.

Parameters

Once you select a card, a series of parameters appear that allow you to describe the pins in your setup and how they are connected. BMS Digital Card has two connectors (X1 and X2) used for digital inputs and two connectors used for digital outputs (X3 and X4).

Table 1 lists available parameters for digital input connector, and Table 2 lists available parameters for digital output connector. The first letter of the property "Name" differs depending on the Slot where the card is located. It can be derived by replacing s with: A, B, C, D, E, F, G, H.

When you select that a pin is connected you will be able to check if you want to Use signal chooser to select a signal from the model that you want to send to the output of that pin. When this is checked as True, the Signal property will appear where you can select the appropriate signal reference. The Invert property will appear next to the Signal property. Check this if the digital signal should be inverted.

Ports

Signals from the inputs of the HIL Connect can be accessed through the following output ports of the HIL Connect Interface component (again, the first letter depends on the Slot where this card is located):

  • sX1
  • sX2

As internal digital probes already exist, these ports can be used for sourcing signals inside of the model. The output of this port is a vector with the length of 8 elements, where each element of that vector corresponds to one pin of connector.

Desired output signals can be sent to the HIL Connect through the following input ports of the HIL Connect Interface component (again, the first letter depends on the Slot where this card is located):

  • sX3
  • sX4

A vector of length 8 should be sent to each input port, where each element of that vector corresponds to one pin of the connector. For example if you want to send a signal to pin 2, you should make sure that that signal is the second element of your vector. If there is a need for inverting signals, inverting them should be done before feeding them to the input terminal.

If signal chooser is used to send measurements to the appropriate output, you should send zero to the input terminal for that pin.
