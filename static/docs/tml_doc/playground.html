Playground

While TML is primarily targeted for running in simulation context, a basic playground with restricted features is supported. Playground code supports only global scope variables, has main_fnc instead of simulation step entry functions and has support for basic console input and output via print(<expr>) and input(<type>) . Mixing playground and simulation code in a single file is not supported. Component definitions must not have main_fnc and must not use print and input . Playground code must not have init_fnc , output_fnc and update_fnc and cannot use model scope (terminals, properties and namespace variables).

print(<expr>) can print values of arbitrary expressions and string literals. input(<type>) can read input for all supported types (numeric and boolean scalars and tensors) except for string types. Note that instead of a value, type declaration is passed as argument.

To use playground, invoke TML compiler for playground context through command line ( tml playground ) and pass the source code file path ( tml playground <path> ). Default executable output file path is the same as for the input file, with extension changed to .out . Different path can be specified using command line parameter --output / -o . Compiled code can be automatically ran using --run / -r compiler flag.

Following snippets show example playground code source files and execution output. 

fn main_fnc():
    print("play1.tml")

    a = input(int)
    b = input(int)

    c = a + b

    print(c)
end

$ tml playground play1.tml --run

int: 2
int: 3

fn main_fnc():
    print("play2.tml")

    test()

    a = input(tensor<int, 3, 1>)
    b = input(tensor<int, 1, 3>)

    c = a + b

    print(c)
end

fn test():
    for i=0:5:
        print("for")
        print(i)
    end

fn main_fnc():
    print("play3.tml")

    a = [[1, 2, 3], [3, 2, 1], [-1, 5, 4]]

    print("min")
    print(min(a))

    print("min!")
    print(min!(a))
end
