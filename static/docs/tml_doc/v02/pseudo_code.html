    Preparation phase

    The following section assumes the symbol table contains sets of definitions only. The preparation phase is responsible to determine places where every variable is modified. The following pseudo-code provides more information about how the preparation phase works starting from the preparation_phase.

    Is one iteration for preparation phase elegant enough

    When encountering a call to an unprocessed function, we could try to process it first before continuing processing the caller. An alternative would be to process function definitions during the first iteration, and introducing a second iteration responsible to process function calls only.

    ST = symbol_table()

def process_scope(AST_nodes):
  scope = ST.create_scope(ST)
  # processing AST nodes corresponding the scope
  for node in AST_nodes.iterate()
    if node.is_var_def():
      process_var_def()
    else if node.is_selection():
      process_selection()
    else if node.is_for_loop():
      process_for()
    else if node.is_while_loop():

    else if node.is_function_call():
      # Could be shifted to the second iteration

def process_function_call(func_call_node):
  func_id = func_call_node.id
  # find the node representing the function definition
  func_def = AST.find_func_def()
  if not func_def.is_processed():
    # Process the function definition if not processed already.
    process_function(func_def)

  mappings = {}
  # Iterate over (arg, param) pairs
  for (arg, param) in func_node.args, func_def.params:
    if arg.is_LHS():                        # in the caller scope
      arg_entry = ST.find_entry(arg)
      param_entry = ST.find_entry(param)
      # Extend the set of arg with all definitions of the param,
      # thus introducing side effects of the call.
      # The `extend` method is actually union.
      arg_entry.extend(param_entry)
    else if arg.is_function_call():
      process_function_call()

    # Create arg:param mappings
    # For convenience keep reference to the symbol table entries
    # corresponding arguments and parameters. 
    # If the arg is not LHS object (e.g. expression or constant), arg_entry
    # is None.
    mappings.add(arg, param, arg_entry, param_entry)

    Consider the following example and the enumeration of nodes. For brevity, enter and exits nodes are omitted.

    a = 5                 # N1

fn main():
  foo(a)              # N2
  foo(a)              # N3
end


fn foo(arg):
  arg += 1            # N4
  b = 3               # N5
  bar(arg, b)         # N6
end


fn bar(arg1, arg2):
  arg1 += 1           # N7
  arg2 *= 2           # N8
end

    Preparation phase starts by processing the global scope, determining the a has been changed by the node N1. Next, we start processing the main that calls foo inside node N2. Since the foo is not present in the symbol table, the process moves to reason about that function. We determine the foo's scope with all variables modified by the foo (arg1, and b respectively). We then encounter on the node N6 representing the function call to the unprocessed function bar, so we switch to processing it. As one may notice after observing the corresponding scope, the bar modifies the arg1 and arg2, respectively.

    After finishing with bar, we continue processing the call to bar inside the node N6. As we may notice, arg is actually changed in the nodes changing arg1 parameter. Thus, we add elements of sets corresponding to arg1 ({N7}) to the set corresponding the arg (currently {N4} only) by applying union between these two sets (the result set {N4, N7}). Similarly, b is modified in the nodes modifying arg2.

    The next step is to finish processing the call to foo at the N2 node. As one may notice, the nodes changing arg also changing the a, thus we use the elements of the set corresponding to arg and appends (union) to the a's set.

    Finally, we process the second call to foo in the node N3. Since the foo has already been processed, it remains to check the redefinitions of a through param arg. Thus, we apply the union on the set corresponding the a and arg. As we may notice, there is not any changes in the result set of a, concluding the preparation phase which outputs the table shown below.

    {
  a: {N1, N4, N7};
  main: {};
  foo: {
    arg: {N4, N7},
    b: {N5, N8}
  }
  bar: {
    arg1: {N7},
    arg2: {N8}
  }
}

    Building CFG Matching The Program Execution

    Before analysing the program behavior, we should determine the CFG matching the program execution. For this purpose, we use a knowledge about how the component is executed on HIL. The first step is to determine (execute/interpret) global definitions of the component. Next, the init function of the component is called to properly set up the component. Afterwards, output and update functions are interchangeably called for the duration of the simulation. From the perspective of the TML compiler, currently able to reason about only one component, the execution of the component is abstracted in the snippet below:

    global_scope_definitions

init()

while true
  output()
  update()

    To build the graph manifested in the previous snippet, the TML compiler connects the last declaration of the global scope with the entry node of the init function's CFG. Afterwards, the compiler generates a while loop with true condition, whose body starts with the entry node belonging to the output function. To ensure the update function is executed after the output, we connect the output's exit node to the update's entry node. We choose to inline functions init, output, and updated instead to introduce calls to them that will be replaced during the process described below. The implementation of connecting global scope with functions init, output, and update, respectively, is present inside the method inter_function_ast_threading of the ASTThreading class.

    Since calls to custom-defined functions (including init, output, and update) behave as macro substitutions, the TML compiler traverses the previously determined CFG and replaces each call with the copy of called function's CFG. # This funciton copies the path starting with the `start_node` and 
# place it immediately after the `pred` node.
# The path usually ends with a node taking one of the foollowing types:
# function exit node, join_if node, for_node, while_node, end_while, and end_for node)
def copy_path(pred, pred2?, start_node):
  curr_node = start_node
  while curr and not cur.is_path_end():         # While node is not the end of a path, process it in the following way
    if curr_node.is_func_call():
      inline_func_call(pred, pred2, curr_node)
    else if curr_node.is_selection(curr_node)
      pred, curr = copy_selection(pred, pred2, curr_node)
    else if curr_node.can_be_jumped():
      # Check if the node has already been visited. If so, stop cycling.
      if curr_node.is_visited:
        reuturn pred
      # The current node belongs to this path and once more that intersects it.
      # One can consider this node as the destination of the jump belonging to the loop. 
      pred, curr, already_processed = copy_jump(pred, pred2, curr_node)
    else:
      # This function copies the `curr`, connects the copy to the `pred`
      # and returns the copy and `curr`'s sucessor as new (pred, curr_node) pair.
      pred, curr = copy_regular_node(pred, curr_node)

  if curr:
    if curr.is_function_exit():
      # copy the function exit node that is considered as the end of path
      curr_copy = COPY_NODE(curr)
      return curr_copy
    else if curr.is_join_if_node():
      # Returning a node preceeding the join node.
      # Check `copy_selection` for more information
      return pred
    else if curr.is_loop_node():
      # The `copy_jump` is reponsible to process this node
      return curr
  else
    return pred


def copy_selection(pred, if_node)
  # The condition preceeding the `if_node` has been already copied
  # as part of the path (done by the `copy_path` function).
  copy_if_node = COPY_NODE(pred, if_node)

  # Nodes at the beginning of the if-branch and else-branch, respectively
  left_succ = if_node.successors[0]
  right_succ = if_node.successors[1]

  # Copy paths matching if and else branches.
  # Note that COPY_PATH returns a node preceeding the join_if_node
  before_join_left = copy_path(copy_if_node, left_succ)
  before_join_right = copy_path(copy_if_node, right_succ)

  # copy the end node and connect with both last nodes of 
  # if and else branches, respectively
  join_if_node = before_join_left.sucessor
  copy_join_if_node = COPY_NODE(join_if_node)
  connect_nodes(before_join_left, copy_join_if_node)
  connect_nodes(before_join_right, copy_join_if_node)

  return copy_join_if_node, join_if_node.successor


def copy_jump(pred, jump_node):
  # Check if the `jump_node` has already been visited.
  if jump_node.is_visited:
    return pred, jump_node

  jump_node.is_visited = true

  # The `pred` comes from the path preceeding the loop.
  # The `jump_node` represents either the first node of 
  # while loop condition, or a node initilizing iteration variable
  # of a for loop.

  # copy the jump node and connect it with predecessor
  copy_jump_node = COPY_NODE(jump_node)
  connect_nodes(pred, copy_jump_node)
  succ = jump_node.successor

  # copy the path (contains condition or a initialization of the iteration variable)
  # until reaching the for_node or a while_node (referred to as loop_node)
  loop_node = copy_path(succ)
  copy_loop_node = COPY_NODE(loop_node)

  # the `loop_node` has two successor, the first successor is at the beginning of the body,
  # while the second represents the end_loop node.
  start_body_node = loop_node.successors[0]
  end_loop = loop_node.successors[1]

  # Calling `copy_path` results in reaching the jump_node again.
  # Since it has already been processed, do not process it twice.
  # Instead, return the last node of the body that jumps to (preceeds) the `jump_node`,
  copy_last_body_node = copy_path(start_body_node, start_body_node)

  # connect the copy_last_body_node with copy_jump_node
  connect_nodes(copy_last_body_node, copy_jump_node)

  copy_end_loop = COPY_NODE(end_loop)
  # connect the `loop_node` with the `copy_end_loop`
  connect_nodes(copy_loop_node, copy_end_loop)

  return copy_end_node, end_node.successor

   # actual copying
   copy_node = COPY_NODE(node)

    Reaching Definition Analysis

    Reaching definition analysis represents the algorithm responsible to determine In/Out sets for every node in the CFG calculated by previous step (check the previous section). The algorithm iteratively solves equations described in Data-Flow Equations section until fixpoint is reached. The following snippet shows the pseudo-code that elaborates about the algorithm

    def RDA(CFG):
    converged = false
    while not converged:
        # Assume the algorithm converges in this iteration.
        converged = true
        for node in CFG.nodes.iterator():
            unchanged = process_node(node)
            # If all nodes remain unchanged, the outer while loop breaks.
            converged = converged AND unchanged


# Solve equations for the `node`
# The return value indicates 
def process_node(node):
    old_in = node.In
    new_in = calc_new_in(node)

    # If the In set remains unchanged, the Out will remain unchanged too
    if old_in == new_in:
        return true

    # store new In set
    node.In = new_in

    kill = node.find_kill_set()
    gen = node.find_gen_set()

    # Calculate the out set of the node
    node.Out = (node.In \ kill) U gen

    return false

# dynamic predecessors.
def calc_new_in(node):
    In = set()
    for pred in node.predecessors:
        In = In U pred.Out

# We need to determine the property of the RHS to be able
# to say if the LHS is static, dynamic, or undefined. 
def process_var_def(node):
    propety = STATIC
    for subexpr_node in node.RHS.expression.subexpressions_nodes():
        subexpr_prop = expr_property(subexpr_node, node.In)
        if subexpr_prop == DYNAMIC:
            property = DYNAMIC
        else if subexpr_prop == UNDEFINED:
            RAISE AN ERROR

    node.LHS.property = propety

# The function checks whether the `expr_node` being part of the expressions
# is static, dynamic, or undefined.
# The `influence_set` represents a set of definition influencing the `expr_node.
def expr_property(expr_node, influence_set):
    if expr_node.is_var():
        # The node is variable that reference the corresponding symbol table entry.
        entry_set = expr_node.symbol_table_entry
        # We apply the intersection of `entry_set` and the `influence_set`.
        intersection = entry_set INTERSECTION influence_set
        # If the `intersection` has no elements, the variable is undefined.
        # If the `intersetion has exactly one element, then variable is static at this point.
        # Otherwise, the variable is dynamic.
        if intersection.len == 0:
            return UNDEFINED
        else if intersection.len == 1:
            return STATIC
        else:
            return DYNAMIC.
