    Code pruning and loop unrolling

    In this phase we shall use DFA information to prune code that is proven unreachable for the given input model. For example, if the condition in the if/else is statically resolvable we can calculate it at compile time and replace this statement with the only block that will ever be executed (either the body of if or else). This serves to support alternative implementation based on user-configurable inputs from the model etc.

    The same principle is applied to loop unrolling where we must unroll loops whose counter variable is marked as must be static, i.e. loops that contain must be static expression in its body which depends on the counter variable.

    ```admonish note A must be static expressions must be evaluated at compile time. For example, TML strings are currently must be static as we shall use them only at compile time, i.e. they should not end-up in the generated code. 

    When the loop counter range is static then we shall replace the loop with its
unrolled version. 

```admonish note
Unrolling shall be done if:
- loop count is static and loop contains _must be static_ expressions in its
  body with loop counter as its part, or
- loop count is static and smaller then configured unrolling threshold.

If a loop count contains _must be static_ where the loop counter is part of the expression but the counter is dynamic, it should be reported as an error.

    Loop unrolling works in tandem with pruning so the inner code shall be pruned on each unroll iteration (because the inner if/else statements can depend on loop counter). Basically, we shall do for loop and if/else interpretation for statically resolvable expressions. This entails that we should recheck static/dynamic property of each inner expression on each iteration during unrolling and unroll/prune inner statements accordingly.

    Also, on each pruning we shall remove code locations of the pruned code from all In/Out sets from DFA and recalculate static/dynamic expressions properties. The rationale for this is that some variable usages might become static after a code pruning step.

    The process of code pruning, unrolling, and static values calculations shall be repeated using the work-list algorithm until nothing more can be statically calculated, pruned or unrolled.

    Code pruning should always be done as soon as possible to avoid unnecessary processing of the code that will eventually be pruned.

    At the end of the process all our if conditions must be dynamic, and all for loop bodies must not contain must be static expressions.
